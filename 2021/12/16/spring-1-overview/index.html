Spring Framework 组成模块概览

Spring Framework 大概由20个模块组成。这些模块可以分为Core Container，Data Access/Integration，Web，AOP（Aspect Oriented Programming），Instrumentation，Messaging 和 Test，如下图（基于Spring 4.3.x）:

Core Container

Core Container 主要包含 spring-core，spring-beans，spring-context，spring-context-support 和 spring-expression（Spring Expression Language）模块 。

spring-core 和 spring-beans 是框架中最基础的部分，包含了IOC和DI（Dependency Injection）。
其中的FactoryBean是用工厂模式实现的，消除了对于编程式单例的需要，分离了逻辑中配置和依赖（解耦的作用）。

  • Core : 主要包含Spring最基本的核心工具类，Core是其他组件的基础核心
  • Beans : 主要包含访问配置文件、创建、管理Bean以及IOC/DI相关的类
  • Context : 继承的Beans的特性,主要为Spring提供 大量的扩展,如:国际化、事件机制、资源加载等，ApplicationContext接口是Context模快的关键
  • SpEL : 模块提供了强大的语言表达式

AOP and Instrumentation

提供了符合AOP Alliance标准的面向切面编程的实现，可以让你定义如方法拦截器和切点，
从而降低程序之间的耦合。

  • AspectJ : 提供了与AspectJ的集成
  • Instrumentation : 提供用于某些应用程序服务器的类工具和类加载器的实现

Messaging

包含了来自Spring Integration项目的关键抽象，如Message、MessageChannel、MessageHandler等，这些抽象可以作为基于消息的应用程序的基础。该模块还包括一组用于将消息映射到方法的注释，类似于基于Spring MVC注释的编程模型。

Data Access/Integration

Data Access/Integration 由 JDBC、ORM、OXM、JMS 和 Transaction 组成。

  • JDBC : 提供了JDBC抽象层，避免了对于繁琐的JDBC编码和数据库厂商特定错误码（不同数据库错误码可能不同）的解析。
  • Transaction : 支持对实现了特定接口的类和所有pojo(Java普通对象)进行编程式和申明式事务管理
  • ORM : 为流行的对象关系映射api(包括JPA、JDO和Hibernate)提供集成层。使用Spring-ORM模块，可以将所有这些O/ R映射框架与Spring提供的所有其他特性结合使用，例如前面提到的简单声明性事务管理特性。
  • OXM : 提供了一个抽象层，支持对象/XML映射实现，如JAXB、Castor、XMLBeans、JiBX和XStream。
  • JMS : 包含用于生成和使用消息的功能。从Spring Framework 4.1开始，提供了与Spring-Messaging模块集成。

Web

由spring-web、spring-webmvc、spring-websocket和spring-webmvc-portlet模块组成。

  • Web : 提供了基本的面向web的集成特性，比如多部分文件上传功能，以及使用Servlet监听器和面向web的应用上下文初始化IoC容器。它还包含HTTP客户端和Spring远程支持的web相关部分。
  • WebMVC(也叫Web-Servlet) : 包含了Spring的模型-视图-控制器(MVC)和Web应用程序的REST Web Services实现。Spring的MVC框架在领域模型代码和web表单之间提供了清晰的分离，并与Spring框架的所有其他特性集成。
  • webmvc-portlet(也叫Web-Portlet) : 提供了在Portlet环境中使用的MVC实现，并反映了基于servlet的spring-webmvc模块的功能。

Test

支持使用JUnit或TestNG对Spring组件进行单元测试和集成测试。它提供了Spring ApplicationContexts的一致加载和这些上下文的缓存。它还提供了模拟对象，您可以使用这些对象单独测试代码。

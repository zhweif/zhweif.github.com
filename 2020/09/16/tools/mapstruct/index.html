Bean映射工具 MapStruct

发表于2020-09-16|更新于2020-09-16|Tools
|字数总计:2.7k|阅读时长:13分钟|阅读量:

简介

MapStruct是一个Java注解处理器，用于构建类型安全的映射类。官方文档

只需要定义一个mapper接口，在接口里面声明所需要的映射方法，在编译期间，MapStruct就会生成这个接口的实现，
这个实现使用Java普通方法调用的方式来映射源对象和目标对象，而不是使用类似反射之类的方式。

与手工编写映射代码相比，MapStruct通过生成冗长且容易出错的代码来节省时间。遵循约定而非配置的方法，MapStruct使用合理的默认值，但在配置或实现特殊行为时，它会自动采取步骤。

与动态映射框架相比，MapStruct有以下优点:

  • 通过使用普通方法调用而不是反射来快速执行
  • 编译时类型安全:只有相互映射的对象和属性才能被映射，订单实体不会意外映射到客户DTO等。
  • 在构建时清除错误报告
    • 映射是不完整的(不是所有的目标属性都被映射)
    • 映射不正确(找不到合适的映射方法或类型转换)

定义映射接口

定义一个接口并在接口上面使用org.mapstruct.Mapper注解标注

@Mapper()注解的常用参数:

Option Purpose Default
mapstruct.defaultComponentModel 基于应该生成的映射器的组件模型的名称:
可选值 :
default : 默认值,不使用组件框架,可以通过MapStruct提供的工厂方法获取接口实现的实例:Mappers#getMapper(Class)
cdi : 生成的映射器是一个应用程序范围的CDI bean，可以通过@Inject检索
spring : 生成的映射器是一个单例作用域的Spring bean，可以通过@Autowired检索
jsr330 : 生成的映射器是用{@code @Named}标注的，可以通过@Inject来检索，例如使用Spring		 default
mapstruct.defaultInjectionStrategy 通过参数在映射器中使用的注入类型。这只用于基于注释的组件模型，如CDI、Spring和JSR 330。
可选值:
field : 依赖项将被注入到字段中
constructor : 将生成构造函数。依赖项将通过构造函数注入。		 field
mapstruct.unmappedTargetPolicy 在映射方法的目标对象的属性没有填充源值的情况下应用的默认报告策略。
可选值:
ERROR: 任何未映射的目标属性都将导致映射代码生成失败
WARN : 任何未映射的目标属性都将在构建时引起警告
IGNORE : 未映射的目标属性将被忽略		 WARN
unmappedSourcePolicy 与unmappedTargetPolicy类似 WARN

如果TargetObject中某个字段名称与SourceObject中的字段名称不一致，可以使用@Mapping(source = “sourceFieldName”, target = “targetFieldName”)指定映射
例如：@Mapping(source = “title”, target = “goodsTitle”)，将Goods中的title字段映射到GoodsDto中的goodsTitle字段
